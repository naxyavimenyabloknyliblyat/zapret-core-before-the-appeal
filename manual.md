# Часть I. Теория

## 1. Зачем нужен пресет

Провайдер (или государственный фильтр, «ТСПУ») смотрит твой интернет-трафик
специальным оборудованием — **DPI** (Deep Packet Inspection, «глубокий анализ
пакетов»). Когда ты заходишь на заблокированный сайт, DPI видит **имя сайта**
прямо в первом пакете и обрывает соединение.

**Обход** — это специально «испортить» этот первый пакет так, что:
- твой компьютер и настоящий сервер друг друга понимают,
- а DPI **не понимает**, какой это сайт, и пропускает соединение.

Этим занимается движок `winws.exe`. **Пресет** — это инструкция движку: *какой
трафик трогать* и *как именно его портить*. ZAPRET CORE уже содержит готовые
профили (P1–P40). Вкладка **User** даёт написать **свой** профиль под своего
провайдера.

> **Определение.** *Пресет* — текстовый файл `custom\*.txt`, где каждая строка —
> один аргумент для `winws.exe`. Программа читает файл, дополняет его и запускает
> движок.

---

## 2. Словарь главных слов

Прежде чем писать, надо понимать 8 слов. Они будут встречаться постоянно.

| Слово | Простыми словами |
|-------|------------------|
| **DPI / ТСПУ** | Оборудование провайдера, которое читает трафик и блокирует сайты. Наш «противник». |
| **winws.exe** | Движок обхода (из проекта [bol-van/zapret](https://github.com/bol-van/zapret)). Делает всю работу, пресет им управляет. |
| **Пакет** | Маленький кусок данных, которым обмениваются компьютер и сайт. Блокировка решается по **первому** пакету соединения. |
| **SNI** | Имя сайта (например `youtube.com`) открытым текстом внутри первого TLS-пакета. Именно его видит DPI. |
| **TLS** | Защищённое соединение (всё, что по `https://`). Идёт по **TCP**, порт **443**. SNI лежит внутри. |
| **QUIC / HTTP3** | Новый быстрый протокол (его любят YouTube/Google). Идёт по **UDP**, порт **443**. Тоже содержит SNI. |
| **desync (десинхронизация)** | Сам приём обхода: «рассинхронизировать» DPI и сервер — послать фейк, разрезать пакет и т.п. |
| **fake (фейк)** | Поддельный пакет, который мы шлём перед настоящим. DPI верит фейку и ошибается, реальный пакет проходит. |

> Запомни две дороги одного сайта:
> - **TCP 443** = TLS (обычный `https`)
> - **UDP 443** = QUIC (быстрый протокол Google/YouTube)
>
> Поэтому в хорошем пресете почти всегда есть **обе** секции — на TCP и на UDP.

---

## 3. Как ZAPRET CORE использует твой файл

Когда ты выбираешь свой пресет и запускаешь обход, программа делает 5 шагов:

1. Читает файл `custom\<твой>.txt` построчно.
2. Выкидывает комментарии (`#…`) и пустые строки.
3. Превращает короткие пути (`bin/…`, `lists/…`) в полные (`C:\…\bin\…`).
4. Приклеивает в начало `winws.exe` и порты `--wf-tcp` / `--wf-udp`.
5. Склеивает всё в одну команду и запускает движок.

Схематично итоговая команда такая:

```
"…\bin\winws.exe"  --wf-tcp=…  --wf-udp=…   <ВСЕ строки из твоего файла, по порядку>
```

**Вывод, который надо усвоить сразу:** ты пишешь **только середину** — фильтры и
стратегии. Начало (`winws.exe`, `--wf-tcp`, `--wf-udp`) добавляется само. Если
напишешь их руками — будет дубль и движок не запустится (подробно — раздел 7).

# Часть II. Практика

## 4. Где лежит файл и как его открыть

Пресеты лежат в папке `custom\` рядом с `zapret core.exe`:

```
zapret core.exe
custom\
   user.txt      ← первый пресет (создаётся сам)
   user2.txt     ← следующий
   user3.txt
```

**Создать новый пресет:**
1. Открой **Settings → Профили**.
2. Сверху выбери вкладку **User**.
3. Внизу списка нажми **➕ Создать новый пресет**.
4. Программа создаст `custom\userN.txt` с рабочим шаблоном и **сразу откроет его в
   блокноте**.

**Открыть существующий для правки:**
- **Двойной клик** по карточке пресета → откроется в блокноте.

**Кнопка «Как сделать пресет? (гайд)»** под списком открывает этот документ в
браузере.

---

## 5. Как сохранять и применять

1. Отредактировал файл в блокноте → **Файл → Сохранить** (Ctrl+S).
   - Кодировка — **UTF-8** (по умолчанию в современном Блокноте уже так).
   - Перенос строк — любой, программа понимает и Windows, и Unix.
2. Вернись в ZAPRET CORE, на вкладке **User** **одним кликом** выбери карточку
   своего пресета (она подсветится — стала активной).
3. Запусти обход (главная кнопка). Если обход уже шёл — он перезапустится с новым
   пресетом на лету.

> Совет: держи **несколько** пресетов (user, user2, …) под разные стратегии и
> переключайся кликом, чтобы быстро найти рабочий.

# Часть III. Анатомия пресета

Эта часть — самая важная. Разбираем пресет по косточкам, чтобы ты понимал **куда**
что писать.

## 6. Строки, комментарии, аргументы

Файл — это просто список строк. Каждая строка — одно из трёх:

```
# Это комментарий — программа его игнорирует. Пиши сюда заметки.

--dpi-desync=fake          ← это АРГУМЕНТ (одна настройка движка)
                           ↑ пустая строка над ним — тоже игнорируется
```

Правила строк:
- **Один аргумент = одна строка.** Нельзя два аргумента в строке, нельзя один
  аргумент на две строки.
- Аргумент это либо **флаг** (`--new`), либо **ключ=значение** (`--dpi-desync=fake`).
- Пробелы по краям строки убираются автоматически.

## 7. Что подставляет программа сама

Программа **сама** добавляет в начало команды:

```
"…\bin\winws.exe"  --wf-tcp=80,443,2053,2083,2087,2096,8443  --wf-udp=443,19294-19344,50000-50100
```

`--wf-tcp` / `--wf-udp` — это «**какие порты вообще перехватывать**». Если включён
**Game Mode**, сюда добавляется ещё `1024-65535`.

### Почему это критично для тебя

Твои фильтры `--filter-tcp=…` / `--filter-udp=…` работают **только внутри** этих
портов. Это как воронка: сначала `--wf-*` решает «эти порты вообще ловим», потом
`--filter-*` выбирает из них нужные для секции.

```
Весь трафик ─▶ [ --wf-tcp/--wf-udp: ловим эти порты ] ─▶ [ --filter-* в секции: берём из них ] ─▶ стратегия
```

**Правило:** порт в `--filter-tcp` должен быть среди
`80, 443, 2053, 2083, 2087, 2096, 8443` (или включи Game Mode для `1024-65535`).
Порт в `--filter-udp` — среди `443, 19294-19344, 50000-50100`.

Если напишешь `--filter-tcp=8888`, а его нет в `--wf-tcp`, секция **никогда не
сработает** — пакет до неё не дойдёт.

> ❌ **Никогда не пиши в файле** `winws.exe`, `--wf-tcp=`, `--wf-udp=` — это дубли,
> движок упадёт.

## 8. Секция = фильтр + стратегия

Пресет состоит из **секций**. Одна секция отвечает за один вид трафика. Внутри
секции — две части, строго в этом порядке:

```
┌─ ФИЛЬТР: какой трафик ловим ─────────────────────┐
│ --filter-udp=443           ← порт/протокол       │
│ --hostlist=lists/...       ← для каких доменов    │
│ --hostlist-exclude=...     ← кого не трогать      │
├─ СТРАТЕГИЯ: что с ним делаем ─────────────────────┤
│ --dpi-desync=fake          ← метод обхода         │
│ --dpi-desync-repeats=6     ← настройка метода     │
│ --dpi-desync-fake-quic=bin/... ← фейковый пакет   │
└──────────────────────────────────────────────────┘
```

**Куда что вставлять — запомни этот порядок:**
1. Сначала **фильтры** (`--filter-*`, `--hostlist*`, `--ipset*`, `--ip-id`).
2. Потом **стратегия** (`--dpi-desync` и все `--dpi-desync-*`).

Фильтры в одной секции складываются по «И»: `--filter-tcp=443` + `--hostlist=…`
значит «TCP 443 **И** домен из списка».

## 9. Разделитель `--new` и порядок секций

Несколько секций склеиваются строкой `--new` между ними:

```
<секция 1>
--new
<секция 2>
--new
<секция 3>
```

- `--new` ставится **между** секциями (не в начале файла и не в конце).
- Пакет идёт по секциям **сверху вниз** и попадает в **первую**, чей фильтр
  подошёл. Поэтому **более узкие** секции (конкретные домены/порты) ставь **выше**,
  а общие («всё остальное по IP») — **ниже**.

```
1) discord.media (узко)      ← выше
2) сайты из списка (шире)
3) всё остальное по IP (широко) ← ниже
```

## 10. Куда и как вставлять пути к файлам

Везде, где аргумент ссылается на файл (список доменов, фейковый пакет), путь
пишется **коротко и через прямой слэш**:

```
✅ --hostlist=lists/list-general.txt
✅ --dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin
```

Программа сама превратит это в полный путь в кавычках:
`"C:\…\lists\list-general.txt"`.

**Как НЕ надо:**
```
❌ --hostlist="lists/list-general.txt"      (свои кавычки не нужны)
❌ --hostlist=lists\list-general.txt         (обратный слэш)
❌ --hostlist=C:\Program\…\lists\file.txt    (полный путь руками)
```

Абсолютизируются только пути, начинающиеся с `bin/`, `lists/`, `fakes/`, `lua/`
(можно с `./`). Всё остальное (домены, IP, hex-числа) пишется как есть.

# Часть IV. Сборка пресета с нуля

Теперь соберём рабочий пресет по шагам — от одной строки к полному файлу. Можешь
повторять в своём `userN.txt`.

## 11. Шаг 1 — минимальная рабочая секция

Самое маленькое, что уже что-то делает — одна секция на TLS-сайты:

```
--filter-tcp=443
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin
```

Читается так: «лови TCP 443, для каждого соединения пошли 6 фейковых TLS-пакетов
из файла `bin/...`». Уже может разблокировать часть сайтов.

## 12. Шаг 2 — добавляем списки доменов

Трогать **весь** трафик 443 вредно (замедляет, ломает рабочие сайты). Ограничим
секцию списком заблокированных доменов и добавим исключения:

```
--filter-tcp=443
--hostlist=lists/list-general.txt           ← только эти домены
--hostlist=lists/list-general-user.txt      ← + твои домены
--hostlist-exclude=lists/list-exclude.txt   ← этих не трогать
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin
```

> **Куда вставлять списки:** строки `--hostlist*` и `--ipset*` идут в **фильтрную**
> часть секции, до `--dpi-desync`. Своих доменов добавляй не сюда, а в файл
> `lists\list-general-user.txt`.

## 13. Шаг 3 — вторая секция

Сайты ходят и по QUIC (UDP 443). Добавим вторую секцию через `--new`. UDP лечится
фейковым **QUIC**-пакетом:

```
# Секция A — QUIC (UDP 443)
--filter-udp=443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# Секция B — TLS (TCP 80,443)
--filter-tcp=80,443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake,fakedsplit
--dpi-desync-repeats=6
--dpi-desync-fooling=ts
--dpi-desync-fakedsplit-pattern=0x00
--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin
```

Обрати внимание: для UDP — `--dpi-desync-fake-quic`, для TCP — `--dpi-desync-fake-tls`.
Это потому что протоколы разные (см. словарь, раздел 2).

## 14. Шаг 4 — голос Discord и игры

Голос Discord и игровой трафик идут по UDP на других портах и лечатся фильтром по
протоколу `--filter-l7`:

```
--new
# Секция C — голос Discord / STUN
--filter-udp=19294-19344,50000-50100
--filter-l7=discord,stun
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-stun=bin/stun.bin
```

> `--filter-l7` ставится в фильтрной части и означает «фильтровать по типу
> протокола», а не по порту. Значения: `http`, `tls`, `quic`, `discord`, `stun`,
> `wireguard`, `dht`, `unknown`.

## 15. Шаг 5 — обход по IP

Иногда домена не видно (соединение сразу по IP). Тогда фильтруем не по hostlist, а
по диапазонам IP — файлом `ipset-all.txt`. Эти секции ставим **в самый низ** (они
самые широкие):

```
--new
# Секция D — всё остальное по IP, UDP
--filter-udp=443
--ipset=lists/ipset-all.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# Секция E — всё остальное по IP, TCP
--filter-tcp=80,443
--ipset=lists/ipset-all.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=syndata
```

## 16. Готовый полный пресет

Соединяем шаги 3–5 — получается полноценный пресет (это и есть шаблон по умолчанию):

```
# 1) QUIC (UDP 443) для сайтов из списка
--filter-udp=443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--ipset-exclude=lists/ipset-exclude-user.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# 2) Голос Discord / STUN
--filter-udp=19294-19344,50000-50100
--filter-l7=discord,stun
--dpi-desync=fake
--dpi-desync-repeats=6
--new
# 3) Сайты TLS (TCP 80,443) — маскировка SNI
--filter-tcp=80,443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--ipset-exclude=lists/ipset-exclude-user.txt
--dpi-desync=hostfakesplit
--dpi-desync-repeats=4
--dpi-desync-fooling=ts,md5sig
--dpi-desync-hostfakesplit-mod=host=ozon.ru
--new
# 4) Google (TCP 443)
--filter-tcp=443
--hostlist=lists/list-google.txt
--ip-id=zero
--dpi-desync=hostfakesplit
--dpi-desync-repeats=4
--dpi-desync-fooling=ts
--dpi-desync-hostfakesplit-mod=host=www.google.com
--new
# 5) Всё остальное по IP (UDP 443)
--filter-udp=443
--ipset=lists/ipset-all.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# 6) Всё остальное по IP (TCP 80,443)
--filter-tcp=80,443
--ipset=lists/ipset-all.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=syndata
```

# Часть V. Справочник

## 17. Справочник аргументов

Для каждого аргумента: **что делает**, **куда вставлять** (в фильтр или в
стратегию) и **с чем сочетать**.

### 17.1 Фильтры — ставятся в начало секции

| Аргумент | Что делает | Контекст |
|----------|------------|----------|
| `--filter-tcp=80,443` | Ловить TCP на портах. Диапазон: `1024-65535`. | Первая строка секции. Порт должен быть в `--wf-tcp` (разд. 7). |
| `--filter-udp=443` | Ловить UDP на портах. | Первая строка секции. Порт должен быть в `--wf-udp`. |
| `--filter-l7=tls,quic` | Фильтр по протоколу, а не порту. Значения: `http, tls, quic, discord, stun, wireguard, dht, unknown`. | Для голоса/игр: `--filter-l7=discord,stun`. |
| `--hostlist-domains=discord.media,ya.ru` | Список доменов прямо в строке (без файла). | Когда доменов 1–3 и неохота заводить файл. |

### 17.2 Списки доменов и IP — тоже в фильтрной части

| Аргумент | Что делает | Контекст |
|----------|------------|----------|
| `--hostlist=lists/list-general.txt` | Секция работает только для доменов из файла. | Можно несколько `--hostlist=` — объединятся. |
| `--hostlist-exclude=lists/list-exclude.txt` | Эти домены НЕ трогать. | Ставь вместе с `--hostlist`. |
| `--ipset=lists/ipset-all.txt` | Фильтр по диапазонам IP (когда домена не видно). | Для «всё остальное» секций. Вместо `--hostlist`. |
| `--ipset-exclude=lists/ipset-exclude.txt` | Эти IP НЕ трогать. | Почти всегда добавляй для стабильности. |
| `--ip-id=zero` | Обнулять IP ID. | Помогает на части провайдеров, часто для Google-секции. |

### 17.3 Метод обхода `--dpi-desync` — первая строка стратегии

| Значение | Идея | Когда брать |
|----------|------|-------------|
| `fake` | Фейк перед настоящим. DPI верит фейку. | Основное для **UDP/QUIC**. |
| `fake,fakedsplit` | Фейк + разрез TCP-пакета. | Хорошее **начало для TLS**. |
| `fake,multisplit` | Фейк + несколько разрезов. | Если `fakedsplit` не зашёл. |
| `fake,multidisorder` | Фейк + разрезы + перестановка. | Самый «злой» для TLS. |
| `hostfakesplit` | Разрез + маскировка SNI под чужой хост. | Когда `fake` не помогает. Нужен `-mod=host=`. |
| `syndata` | Подмена данных в TCP SYN. | Против блокировки **по IP**. |
| `split` / `split2` / `disorder` / `disorder2` | Просто разрезать / переставить (без фейка). | Альтернативы для экспериментов. |

### 17.4 Настройка метода — после `--dpi-desync`

| Аргумент | Что делает | Контекст |
|----------|------------|----------|
| `--dpi-desync-repeats=6` | Сколько раз повторить (надёжнее, но шумнее). | Разумно 4–16. Главная «крутилка». |
| `--dpi-desync-fooling=ts,badseq,md5sig` | Чем «портить» фейк, чтобы он не дошёл до сервера, но обманул DPI. Значения: `ts, badseq, badsum, md5sig, datanoack`. | Только для `fake`-методов. |
| `--dpi-desync-autottl=2` | Авто-подбор TTL (фейк гаснет до сервера). Усиливает. | Параметр — «дельта», обычно 1–3. |
| `--dpi-desync-ttl=4` | Жёсткий TTL вместо autottl. | Если autottl не подошёл. |
| `--dpi-desync-split-pos=1,midsld` | Где резать пакет. | Для `*split*`/`*disorder*`. |
| `--dpi-desync-fakedsplit-pattern=0x00` | Чем заполнять фейковую часть. | Для `fakedsplit`. |
| `--dpi-desync-hostfakesplit-mod=host=ozon.ru` | Под какой хост маскироваться. | Обязателен для `hostfakesplit`. Часто: `ozon.ru`, `www.google.com`, `ya.ru`. |
| `--dpi-desync-any-protocol=1` | Применять к любому протоколу на порту. | Для секций «накрыть всё по IP». |
| `--dpi-desync-cutoff=n3` | До какого пакета работать (`n2,n3,n4,d2…`). | Экономит ресурсы в широких секциях. |

### 17.5 Фейковые пакеты `--dpi-desync-fake-*` — в конце стратегии

Подбираются **под тип трафика секции**:

| Аргумент | Для чего |
|----------|----------|
| `--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin` | UDP 443 (QUIC). |
| `--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin` | TCP 443 (TLS). |
| `--dpi-desync-fake-http=bin/tls_clienthello_max_ru.bin` | TCP 80 (HTTP). |
| `--dpi-desync-fake-stun=bin/stun.bin` | UDP голос/игры (STUN). |
| `--dpi-desync-fake-unknown-udp=bin/quic_initial_www_google_com.bin` | Неизвестный UDP (Game Mode). |
| `--dpi-desync-fake-discord=bin/quic_initial_dbankcloud_ru.bin` | Трафик Discord. |

> Можно указать **несколько** `--dpi-desync-fake-tls=` подряд с разными файлами —
> движок пошлёт несколько разных фейков. Иногда это решает там, где один не заходит.

### 17.6 Разделитель

| Аргумент | Что делает |
|----------|------------|
| `--new` | Завершает текущую секцию и начинает новую. Ставится **между** секциями. |

# Часть VI. Поддержка

## 20. Если не работает — алгоритм

Меняй **по одному параметру** и проверяй, иначе не поймёшь, что помогло.

1. **Сайты по `https` не грузятся** → в TLS-секции пробуй стратегии по очереди:
   - `--dpi-desync=fake,fakedsplit` + `--dpi-desync-fakedsplit-pattern=0x00`
   - `--dpi-desync=fake,multisplit` + `--dpi-desync-split-pos=1`
   - `--dpi-desync=fake,multidisorder` + `--dpi-desync-fooling=badseq`
   - `--dpi-desync=hostfakesplit` + `--dpi-desync-hostfakesplit-mod=host=ozon.ru`
2. **`fake` не помогает** → смени файл в `--dpi-desync-fake-tls=` (перебери `.bin`),
   добавь `--dpi-desync-autottl=2`.
3. **Меняй маскировку хоста:** `host=ozon.ru` / `host=www.google.com` / `host=ya.ru`.
4. **YouTube/Discord рвётся по QUIC** → увеличь `--dpi-desync-repeats` в UDP-секциях
   **или** отключи QUIC/HTTP3 в браузере (тогда трафик пойдёт по TCP и сработают
   TLS-секции).
5. **Сайт ходит по IP, а не по домену** → перенеси правило в секцию с
   `--ipset=lists/ipset-all.txt`.
6. **Совсем ничего** → проверь, что порт из `--filter-*` входит в авто-`--wf-*`
   (раздел 7). Если нет — включи Game Mode или поменяй порт.

## 21. Если хочешь жёстче / крепче

Помогает против сильного ТСПУ, но растёт нагрузка и «шум», может просесть скорость
и пинг. Усиливай **постепенно**, до первого стабильного результата.

- **Больше повторов:** `--dpi-desync-repeats=10 … 16`.
- **Авто-TTL в каждую desync-секцию:** `--dpi-desync-autottl=2`.
- **Усиленный обман:** `--dpi-desync-fooling=ts,badseq,md5sig`.
- **Несколько фейков сразу:** продублируй `--dpi-desync-fake-tls=` с разными `.bin`.
- **Самый агрессивный TLS:**
  ```
  --dpi-desync=fake,multidisorder
  --dpi-desync-split-pos=1,midsld
  --dpi-desync-fooling=badseq,md5sig
  --dpi-desync-repeats=12
  --dpi-desync-autottl=2
  ```
- **Накрыть весь трафик по IP (нужен Game Mode для портов `1024-65535`):**
  ```
  --new
  --filter-tcp=1024-65535
  --ipset=lists/ipset-all.txt
  --ipset-exclude=lists/ipset-exclude.txt
  --dpi-desync=fake,fakedsplit
  --dpi-desync-repeats=8
  --dpi-desync-any-protocol=1
  --dpi-desync-cutoff=n4
  ```

## 22. Ошибки формата

| Симптом | Причина | Решение |
|---------|---------|---------|
| Движок не стартует / падает сразу | В файле написан `winws.exe` или `--wf-tcp/--wf-udp` (дубль) | Убери их — программа подставляет сама (разд. 7) |
| Обход «включён», но ничего не разблокировано | Порт `--filter-*` не входит в авто-`--wf-*` | Используй порты из разд. 7 или включи Game Mode |
| Путь «не найден» | Путь через `\`, в кавычках или полным руками | Пиши `bin/…` и `lists/…` через `/`, без кавычек (разд. 10) |
| Секция игнорируется | Аргумент разбит на две строки | Весь `--ключ=значение` — на одной строке |
| Строка «пропала» | Перед ней стоит `#` | `#` = комментарий; убери, если строка нужна |
| Кракозябры в комментариях | Файл сохранён не в UTF-8 | Пересохрани как UTF-8 |
| Конкретный домен не разблокировался | Его нет в списках | Добавь в `lists\list-general-user.txt` |
| `hostfakesplit` не работает | Нет `--dpi-desync-hostfakesplit-mod=host=…` | Добавь маскировку хоста |

## 23. Чеклист перед запуском

- [ ] Нет строк с `winws.exe`, `--wf-tcp`, `--wf-udp`.
- [ ] Все пути — через `/`, с префиксом `bin/` или `lists/`, без кавычек.
- [ ] В каждой секции сначала фильтры, потом `--dpi-desync*`.
- [ ] Секции разделены `--new` (между секциями, не в начале/конце).
- [ ] Порты `--filter-tcp`/`--filter-udp` входят в авто-`--wf-*` (или включён Game Mode).
- [ ] У каждого `--dpi-desync-fake-*` указан существующий файл из `bin\`.
- [ ] Узкие секции выше, широкие (`ipset-all`) — ниже.
- [ ] Файл сохранён в UTF-8.
- [ ] Параметры меняешь по одному и проверяешь результат.
