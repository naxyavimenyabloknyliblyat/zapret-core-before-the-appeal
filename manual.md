# ZAPRET CORE — Полное руководство по созданию пресетов (вкладка **User**)

> Этот документ объясняет, **как правильно написать свой пресет** для движка
> `winws.exe` через вкладку **User** в ZAPRET CORE, чтобы программа его приняла
> и не выдала ошибку формата, а сам обход реально заработал.
>
> Читается долго, но один раз — после этого ты сможешь собирать любые стратегии
> обхода DPI под своего провайдера.

## 1. Что такое пресет

**Пресет** — это текстовый файл с набором аргументов для `winws.exe` (это движок
из проекта [bol-van/zapret](https://github.com/bol-van/zapret), который и делает
обход DPI). Каждый штатный профиль программы (P1, P2, … P40) — это такой же набор
аргументов, просто зашитый в код. Вкладка **User** позволяет написать **свой**
набор и запускать его как обычный профиль.

Файл пресета НЕ запускается напрямую. ZAPRET CORE:
1. читает твой файл построчно,
2. отбрасывает комментарии и пустые строки,
3. превращает относительные пути (`bin/...`, `lists/...`) в абсолютные,
4. подставляет в начало `winws.exe` и порты `--wf-tcp` / `--wf-udp`,
5. склеивает всё в одну командную строку и запускает движок.

Поэтому формат файла должен быть **ровно таким, как ожидает парсер** — иначе
получишь либо ошибку, либо «обход вроде включён, но ничего не разблокировалось».

## 2. Где лежит файл и как его создать

Пресеты лежат в папке `custom\` рядом с `zapret core.exe`:

```
zapret core.exe
custom\
  user.txt      ← первый пресет (создаётся автоматически)
  user2.txt     ← следующий
  user3.txt
  ...
```

**Как создать новый:**
- Открой **Settings → Профили → вкладка User**.
- Внизу списка нажми **➕ Создать новый пресет**.
- Программа создаст `custom\userN.txt` с рабочим шаблоном и **сразу откроет его в
  блокноте**. Имя файла можно увидеть на карточке.

**Как редактировать:**
- **Двойной клик** по карточке пресета — откроется в блокноте.

**Как выбрать активный:**
- **Одинарный клик** по карточке — пресет становится выбранным (как обычный
  профиль). Если обход уже запущен — перезапустится на лету.

**Сохранение:**
- Файл сохраняй как обычный `.txt` в кодировке **UTF-8**. Перенос строк — любой
  (Windows `CRLF` или Unix `LF`), парсер понимает оба.

## 3. Железные правила формата

Эти правила — самое важное в документе. Нарушишь — будет ошибка или тихо не
сработает.

| # | Правило | Пример |
|---|---------|--------|
| 1 | **Один аргумент = одна строка.** | `--dpi-desync=fake` на отдельной строке |
| 2 | **Комментарии начинаются с `#`** и игнорируются целиком. | `# это комментарий` |
| 3 | **Пустые строки** разрешены, игнорируются. | |
| 4 | **Пути пиши через `/`** и без кавычек. Программа сама добавит кавычки и сделает путь абсолютным. | `--hostlist=lists/list-general.txt` |
| 5 | **НЕ пиши `winws.exe`** в начале — он подставляется сам. | ❌ `winws.exe --dpi-desync=...` |
| 6 | **НЕ пиши `--wf-tcp` / `--wf-udp`** — они подставляются сами. Дубликат ломает запуск. | ❌ `--wf-tcp=80,443` |
| 7 | Абсолютизируются только пути с префиксами `bin/`, `lists/`, `fakes/`, `lua/` (можно с `./`). Остальное (домены, IP, hex) — как есть. | `bin/stun.bin` → `"C:\...\bin\stun.bin"` |
| 8 | **Каждая секция отделяется `--new`** (об этом — раздел 5). | |
| 9 | Не разбивай один аргумент на две строки. Значение `--key=value` целиком на одной строке. | ❌ `--dpi-desync=` \ `fake` |
| 10 | Хвостовой `^` (перенос из `.bat`) срезается автоматически, но лучше его не писать. | |

> **Главное, что вызывает «не тот формат» / нерабочий обход:**
> писать `winws.exe`, `--wf-tcp`, `--wf-udp` руками (правила 5–6),
> ставить кавычки вокруг путей вручную, или писать пути через `\` (правило 4).

## 4. Что программа подставляет САМА

Когда ты запускаешь свой пресет, ZAPRET CORE формирует команду так:

```
"C:\...\bin\winws.exe" --wf-tcp=<порты> --wf-udp=<порты>  <твои строки из файла>
```

`--wf-tcp` / `--wf-udp` — это «какие порты вообще перехватывать на уровне
WinDivert». По умолчанию (Game Mode выключен):

```
--wf-tcp=80,443,2053,2083,2087,2096,8443
--wf-udp=443,19294-19344,50000-50100
```

Если включён **Game Mode**, к соответствующему списку добавляется `1024-65535`
(TCP, UDP или оба — зависит от фильтра Game Mode).

### Почему это важно для тебя

Твои `--filter-tcp=` / `--filter-udp=` в секциях **работают только в пределах
этих `--wf-*` портов**. Если напишешь `--filter-tcp=8888`, а порта `8888` нет в
`--wf-tcp`, то секция просто никогда не сработает — пакеты на 8888 даже не будут
перехвачены.

**Вывод:** в `--filter-tcp` используй порты из набора
`80,443,2053,2083,2087,2096,8443` (или включи Game Mode для `1024-65535`),
а в `--filter-udp` — из `443,19294-19344,50000-50100`.

## 5. Структура пресета: секции и `--new`

Пресет состоит из одной или нескольких **секций**. Секция — это:

```
<фильтры: какой трафик ловим>
<стратегия: что с ним делаем>
```

Секции разделяются строкой `--new`. Пакет проходит по секциям сверху вниз и
обрабатывается **первой секцией, чей фильтр под него подошёл**.

```
# Секция 1 — ловим UDP 443 (QUIC) для сайтов из списка
--filter-udp=443
--hostlist=lists/list-general.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# Секция 2 — ловим TCP 80,443 (HTTP/TLS) для тех же сайтов
--filter-tcp=80,443
--hostlist=lists/list-general.txt
--dpi-desync=fake,fakedsplit
--dpi-desync-repeats=6
--dpi-desync-fooling=ts
```

Порядок аргументов внутри секции: **сначала фильтры** (`--filter-*`, `--hostlist*`,
`--ipset*`, `--ip-id`), **потом стратегия** (`--dpi-desync*`). Так читается легче и
совпадает со штатными профилями.

> Типичная раскладка по секциям:
> 1. QUIC (UDP 443) для сайтов
> 2. Голос/игры (UDP игровые порты, `--filter-l7=discord,stun`)
> 3. HTTP/TLS (TCP 80,443) для сайтов
> 4. Отдельная секция для Google
> 5. Всё остальное по IP (`--ipset=lists/ipset-all.txt`)

## 6. Справочник аргументов

### 6.1 Фильтры — что ловим

| Аргумент | Что делает |
|----------|------------|
| `--filter-tcp=80,443` | Ловить TCP на этих портах. Диапазоны: `1024-65535`. |
| `--filter-udp=443` | Ловить UDP на этих портах. |
| `--filter-l7=tls` | Фильтр по протоколу прикладного уровня. Значения: `http`, `tls`, `quic`, `discord`, `stun`, `wireguard`, `dht`, `unknown`. Можно списком: `--filter-l7=discord,stun`. |
| `--hostlist-domains=discord.media` | Применять секцию только к этим доменам (прямо в строке, без файла). Список через запятую. |

Фильтры в одной секции комбинируются по «И»: `--filter-tcp=443` +
`--hostlist=...` = «TCP 443 И домен из списка».

### 6.2 Списки доменов и IP

| Аргумент | Что делает |
|----------|------------|
| `--hostlist=lists/list-general.txt` | Обход только для доменов из файла. Можно указать несколько `--hostlist=` — они объединяются. |
| `--hostlist-exclude=lists/list-exclude.txt` | Исключения — эти домены НЕ трогать. |
| `--ipset=lists/ipset-all.txt` | Обход по диапазонам IP (когда домен не виден, например по IP-адресу). |
| `--ipset-exclude=lists/ipset-exclude.txt` | Исключения по IP. |
| `--ip-id=zero` | Обнулять поле IP ID (помогает на части провайдеров, часто для Google). |

> Свои домены добавляй в `lists\list-general-user.txt`, свои IP-исключения — в
> `lists\ipset-exclude-user.txt`. Эти файлы для того и существуют.

### 6.3 Методы обхода `--dpi-desync`

Главный аргумент. Определяет **сам приём** обмана DPI. Можно комбинировать через
запятую (`fake,fakedsplit`).

| Значение | Идея |
|----------|------|
| `fake` | Шлём «фейковый» пакет перед настоящим. DPI принимает решение по фейку, реальный запрос проходит мимо блокировки. Для UDP/QUIC — основной метод. |
| `fakedsplit` | Фейк + режем реальный TCP-пакет на части так, что DPI не собирает SNI. |
| `fakeddisorder` | То же, но части идут в «неправильном» порядке. |
| `split` / `split2` | Просто разрезать пакет (без фейка). `split2` — другой способ нарезки. |
| `disorder` / `disorder2` | Разрезать и переставить части. |
| `multisplit` | Несколько точек разреза (надёжнее против умного DPI). |
| `multidisorder` | Несколько разрезов + перестановка. Самый «злой» для TLS. |
| `syndata` | Подмена данных в TCP SYN. Хорошо против блокировки по IP. |
| `hostfakesplit` | Режем и маскируем SNI под чужой разрешённый хост (см. `-mod=host=`). |

### 6.4 Настройка метода

| Аргумент | Что делает |
|----------|------------|
| `--dpi-desync-repeats=6` | Сколько раз повторить фейк/десинк. Больше = надёжнее, но больше «шума» и нагрузки. Разумно 4–16. |
| `--dpi-desync-fooling=ts` | Чем «портить» фейк, чтобы он не дошёл до сервера, но обманул DPI. Значения: `ts`, `badseq`, `badsum`, `md5sig`, `datanoack`. Можно списком: `ts,badseq,md5sig`. |
| `--dpi-desync-autottl=2` | Авто-подбор TTL: фейк гаснет где-то до сервера. Усиливает обход. Параметр — «дельта». |
| `--dpi-desync-ttl=4` | Жёстко задать TTL фейку (вместо autottl). |
| `--dpi-desync-split-pos=1` | Где резать пакет. Можно `1`, `midsld`, список: `1,midsld`. |
| `--dpi-desync-fakedsplit-pattern=0x00` | Чем заполнять фейковую часть при `fakedsplit`. |
| `--dpi-desync-hostfakesplit-mod=host=ozon.ru` | Под какой хост маскироваться при `hostfakesplit`. Частые: `ozon.ru`, `www.google.com`, `ya.ru`. |
| `--dpi-desync-any-protocol=1` | Применять к любому протоколу на порту (для «накрыть всё по IP»). |
| `--dpi-desync-cutoff=n3` | До какого пакета в потоке работать (`n2`,`n3`,`n4`,`d2`...). Экономит ресурсы. |

### 6.5 Фейковые пакеты

Это «тело» фейкового пакета — файл из папки `bin\`. Подбирается под тип трафика:

| Аргумент | Для чего |
|----------|----------|
| `--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin` | Фейковый QUIC (UDP 443). |
| `--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin` | Фейковый TLS ClientHello (TCP 443). |
| `--dpi-desync-fake-http=bin/tls_clienthello_max_ru.bin` | Фейковый HTTP (TCP 80). |
| `--dpi-desync-fake-stun=bin/stun.bin` | Фейковый STUN (голос/игры по UDP). |
| `--dpi-desync-fake-unknown-udp=bin/quic_initial_www_google_com.bin` | Фейк для неизвестного UDP (Game Mode). |
| `--dpi-desync-fake-discord=bin/quic_initial_dbankcloud_ru.bin` | Фейк для трафика Discord. |

> Можно указать **несколько** `--dpi-desync-fake-tls=` подряд с разными файлами —
> движок отправит несколько разных фейков. Иногда это решает там, где один не
> заходит.

## 7. Доступные файлы

### Папка `lists\` (списки)

| Файл | Что внутри |
|------|------------|
| `list-general.txt` | Основной список заблокированных доменов. |
| `list-general-user.txt` | **Твой** список доменов (добавляй сюда). |
| `list-google.txt` | Домены Google/YouTube. |
| `list-exclude.txt` | Домены-исключения. |
| `list-exclude-user.txt` | **Твои** исключения по доменам. |
| `ipset-all.txt` | Диапазоны IP для обхода по адресу. |
| `ipset-exclude.txt` | IP-исключения. |
| `ipset-exclude-user.txt` | **Твои** IP-исключения. |

### Папка `bin\` (фейковые пакеты)

Частые payload-файлы (наличие зависит от сборки):

```
quic_initial_www_google_com.bin
quic_initial_dbankcloud_ru.bin
tls_clienthello_www_google_com.bin
tls_clienthello_max_ru.bin
tls_clienthello_4pda_to.bin
stun.bin
```

> Загляни в свою папку `bin\` — используй те `.bin`, что реально есть. Если файла
> нет, секция с ним работать не будет.

## 8. Готовые рецепты

### 8.1 Базовый (универсальный, для сайтов)

```
# 1) QUIC (UDP 443)
--filter-udp=443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
# 2) HTTP/TLS (TCP 80,443)
--filter-tcp=80,443
--hostlist=lists/list-general.txt
--hostlist=lists/list-general-user.txt
--hostlist-exclude=lists/list-exclude.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake,fakedsplit
--dpi-desync-repeats=6
--dpi-desync-fooling=ts
--dpi-desync-fakedsplit-pattern=0x00
--dpi-desync-fake-tls=bin/tls_clienthello_www_google_com.bin
```

### 8.2 Маскировка SNI (когда `fake` не заходит)

```
--filter-tcp=80,443
--hostlist=lists/list-general.txt
--hostlist-exclude=lists/list-exclude.txt
--dpi-desync=hostfakesplit
--dpi-desync-repeats=4
--dpi-desync-fooling=ts,md5sig
--dpi-desync-hostfakesplit-mod=host=ozon.ru
```

### 8.3 Голос Discord и игры (UDP)

```
--filter-udp=19294-19344,50000-50100
--filter-l7=discord,stun
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-stun=bin/stun.bin
```

### 8.4 Всё остальное по IP (когда домен не в списке)

```
--filter-udp=443
--ipset=lists/ipset-all.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=fake
--dpi-desync-repeats=6
--dpi-desync-fake-quic=bin/quic_initial_www_google_com.bin
--new
--filter-tcp=80,443
--ipset=lists/ipset-all.txt
--ipset-exclude=lists/ipset-exclude.txt
--dpi-desync=syndata
```

> Эти блоки можно соединять через `--new` в один большой пресет (как штатный P3).

## 9. Если не работает — что менять

Меняй **по одному параметру за раз** и проверяй, иначе не поймёшь, что помогло.

1. **Сайты не открываются по HTTPS** → меняй стратегию в TLS-секции по очереди:
   - `--dpi-desync=fake,fakedsplit` + `--dpi-desync-fakedsplit-pattern=0x00`
   - `--dpi-desync=fake,multisplit` + `--dpi-desync-split-pos=1`
   - `--dpi-desync=fake,multidisorder` + `--dpi-desync-fooling=badseq`
   - `--dpi-desync=hostfakesplit` + `--dpi-desync-hostfakesplit-mod=host=ozon.ru`
2. **`fake` не помогает** → добавь/смени `--dpi-desync-fake-tls=bin/...` (попробуй
   разные `.bin`), добавь `--dpi-desync-autottl=2`.
3. **YouTube/Discord рвётся по QUIC** → увеличь `--dpi-desync-repeats` в
   UDP-секциях, или **отключи QUIC/HTTP3 в браузере** — тогда трафик пойдёт по TCP
   и заработают TLS-секции.
4. **Меняй маскировку хоста**: `host=ozon.ru` / `host=www.google.com` / `host=ya.ru`.
5. **Сайт по IP, а не по домену** → перенеси правило в секцию с
   `--ipset=lists/ipset-all.txt`.
6. **Совсем ничего** → проверь, что порт из `--filter-*` входит в авто-`--wf-*`
   (раздел 4). Если нет — включи Game Mode или поменяй порт.

## 10. Если хочешь жёстче / крепче

Полезно против сильного ТСПУ, но растёт нагрузка и «шум», может просесть скорость.

- **Больше повторов:** `--dpi-desync-repeats=10 … 16`.
- **Авто-TTL в каждую desync-секцию:** `--dpi-desync-autottl=2`.
- **Усиленный обман:** `--dpi-desync-fooling=ts,badseq,md5sig`.
- **Несколько фейков сразу:** продублируй `--dpi-desync-fake-tls=` с разными `.bin`.
- **Самый агрессивный TLS:**
  ```
  --dpi-desync=fake,multidisorder
  --dpi-desync-split-pos=1,midsld
  --dpi-desync-fooling=badseq,md5sig
  --dpi-desync-repeats=12
  --dpi-desync-autottl=2
  ```
- **Накрыть весь трафик по IP (как Game Mode):**
  ```
  --new
  --filter-tcp=1024-65535
  --ipset=lists/ipset-all.txt
  --ipset-exclude=lists/ipset-exclude.txt
  --dpi-desync=fake,fakedsplit
  --dpi-desync-repeats=8
  --dpi-desync-any-protocol=1
  --dpi-desync-cutoff=n4
  ```
  (для секции на `1024-65535` нужен включённый Game Mode, иначе порт не попадёт в
  `--wf-tcp`).

> ⚠️ Не задирай `repeats` и порты «на всякий случай» — это бьёт по скорости и пингу.
> Усиливай постепенно, до первого стабильного результата.

## 11. Частые ошибки

| Симптом | Причина | Решение |
|---------|---------|---------|
| Обход «включён», но ничего не разблокировано | Порт `--filter-*` не входит в авто-`--wf-*` | Используй порты из раздела 4 или включи Game Mode |
| Движок не стартует / падает сразу | Написал `winws.exe` или `--wf-tcp/--wf-udp` в файле (дубликат) | Убери их — программа подставляет сама |
| Путь к `.bin`/списку «не найден» | Путь через `\`, в кавычках, или нестандартный префикс | Пиши `bin/...` и `lists/...` через `/`, без кавычек |
| Секция игнорируется | Аргумент разбит на две строки | Весь `--key=value` — на одной строке |
| Часть строк «пропала» | Перед строкой стоит `#` | `#` = комментарий; убери, если строка нужна |
| Кракозябры в комментариях | Файл сохранён не в UTF-8 | Пересохрани как UTF-8 |
| Домен не разблокировался | Его нет в `list-general.txt` | Добавь в `lists\list-general-user.txt` |

## 12. Чеклист перед запуском

- [ ] Нет строк с `winws.exe`, `--wf-tcp`, `--wf-udp`.
- [ ] Все пути — через `/`, с префиксом `bin/` или `lists/`, без кавычек.
- [ ] Каждая секция разделена `--new`.
- [ ] Порты в `--filter-tcp` / `--filter-udp` входят в авто-`--wf-*` (или включён Game Mode).
- [ ] У каждого `--dpi-desync-fake-*` указан реально существующий файл из `bin\`.
- [ ] Файлы списков существуют в `lists\`.
- [ ] Файл сохранён в кодировке UTF-8.
- [ ] Меняешь параметры по одному и проверяешь результат.
